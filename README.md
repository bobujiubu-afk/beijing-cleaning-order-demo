# 北京东科保洁接单本

这是一个给个人老板用的轻量接单本。适合保洁老板、家电维修师傅、空调清洗师傅、搬家师傅、上门服务师傅自己记录客户电话、地址、上门时间和金额。

核心用法很简单：

- 客户没说清楚：把预约链接发给客户填写。
- 微信已经聊清楚：老板自己点“新增一单”。
- 每天干活前：打开“今天的活”。
- 干完后：把状态改成“已完成”。

## 文件作用

- `app.py`：后端主程序，负责预约提交、接单本、统计、Excel 导出、演示数据。
- `templates/submit.html`：客户预约页。
- `templates/admin.html`：老板接单本页面。
- `templates/stats.html`：简单统计页。
- `templates/exports.html`：Excel 导出结果页。
- `templates/base.html`：顶部导航和页面基础结构。
- `static/style.css`：页面样式，兼容手机和电脑。
- `static/admin.js`：接单本自动刷新、保存、消息提醒。
- `static/manifest.webmanifest`：手机桌面入口配置。
- `database.db`：SQLite 数据库，第一次运行后自动生成。
- `requirements.txt`：需要安装的 Python 依赖。
- `start-demo.cmd`：本机一键启动脚本。
- `smoke_test.py`：基础流程自测脚本。

## 本地运行

第一次运行前，在项目目录安装依赖：

```powershell
python -m pip install -r requirements.txt
```

如果这台电脑的 `python` 不可用，可以用已验证的 Python：

```powershell
& "C:\Users\申\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe" -m pip install -r requirements.txt
```

启动：

```powershell
python app.py
```

或者直接双击：

```text
start-demo.cmd
```

浏览器打开：

- 客户预约页：http://127.0.0.1:5000/submit
- 老板接单本：http://127.0.0.1:5000/admin
- 统计页：http://127.0.0.1:5000/stats

## 登录

默认演示密码：

```text
123456
```

真实给别人用前，请在部署平台设置：

```text
ADMIN_PASSWORD=你的新密码
SECRET_KEY=一串随机长字符
```

## 接单本字段

页面只保留个人老板每天真正会用到的字段：

- 客户姓名
- 联系电话
- 上门地址
- 服务项目
- 预约日期
- 预约时间段
- 面积/数量
- 订单金额
- 状态
- 来源
- 备注

状态只有四个：

- 待联系：客户刚提交，还没确认。
- 已约好：已经和客户说好时间、地址、价格。
- 已完成：活已经干完。
- 已取消：客户不要了，或者不做了。

旧数据会自动兼容：

- 已联系、已报价会显示成已约好。
- 已成交会显示成已完成。

## 新增一单

老板在微信里已经聊清楚客户信息时，不需要再让客户填表，直接在 `/admin` 点“新增一单”。

默认值：

- 状态：已约好
- 来源：微信沟通
- 备注：无

保存后会回到“今天的活”。如果预约日期是今天，会直接显示在今天列表里。

## 电话和导航

- 电话使用 `tel:` 链接，手机上可以直接拨打。
- 地址旁边有“导航”按钮。
- 手机点导航会先尝试打开高德 App；如果系统不允许或没安装高德，再退回高德网页。
- 地址不含“北京”时，会自动用“北京 + 地址”去高德地图搜索。

## Excel 导出

统计页可以导出 Excel。导出字段保持简单：

- 客户姓名
- 电话
- 地址
- 服务项目
- 预约日期
- 预约时间段
- 面积/数量
- 订单金额
- 状态
- 来源
- 备注
- 创建时间

不会导出负责人、回访状态、报价金额、成交金额这些复杂字段。

## 消息提醒

老板接单本打开时，会每 5 秒检查一次新客户。如果页面打开后有客户提交预约，会：

- 顶部出现新客户提醒。
- 待联系客户显示红点。
- 新客户排在列表上方。
- 浏览器标题显示“【新客户】北京东科保洁接单本”。
- 开启页面声音后，页面打开时会有提示音和震动。
- 开启手机消息推送后，支持的手机可以收到系统通知。

手机推送设置在右上角齿轮里。手机想收到系统通知，建议这样做：

1. 用 Safari 或 Chrome 打开线上老板接单本，不要用微信内置浏览器。
2. 把页面添加到手机桌面。
3. 从桌面图标打开。
4. 登录后点右上角齿轮。
5. 点“开启手机消息推送”，允许通知权限。
6. 点“发送测试推送”，确认手机通知栏能收到。

网页提醒不能保证和微信原生消息一样 100% 常驻。浏览器关闭、系统省电、通知权限被拒绝、免费服务器休眠时，提醒可能延迟或失败。以后要更接近微信来消息，可以升级微信小程序订阅消息、公众号模板消息或企业微信提醒。

## 数据安全

- 客户预约页 `/submit` 可以公开。
- `/admin`、`/stats`、订单接口、导出、备份都需要登录。
- 后台页面和接口设置了 `Cache-Control: no-store`，减少客户数据被浏览器缓存。
- 登录失败多次会临时锁定。
- 真实使用一定要改 `ADMIN_PASSWORD` 和 `SECRET_KEY`。

## 数据备份

统计页可以下载数据库备份。正式使用时建议每天备份一次。

不要随便删除 `database.db`，删除后本地数据会丢。

如果没有自动部署，可以到 Render 里点：

```text
Manual Deploy -> Deploy latest commit
```

部署后测试：

- `/submit` 客户预约页
- `/admin` 老板接单本
- `/stats` 统计页
- `/export` Excel 导出

如果手机上还是旧样式，先强制刷新；如果是桌面图标打开的，关闭后重新从桌面图标进入。
